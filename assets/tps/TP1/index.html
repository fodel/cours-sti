Contexte du projet :

Afin de lutter contre la grippe saisonnière et d'améliorer la couverture vaccinale, le ministère de la Santé prévoit de lancer un site web dédié à la réservation des vaccins. Cette plateforme intuitive vise à faciliter l'accès à la vaccination pour tous les citoyens.

Objectifs du site web :
  1. Réservation d'un vaccin : Après avoir renseigné quelques informations personnelles, l'utilisateur pourra choisir un centre de vaccination, sélectionner une date et réserver sa dose.
  2. Suivi des réservations : L'utilisateur aura la possibilité de consulter l'état de ses réservations et d'accéder aux détails de ses rendez-vous.
Étape 1: Création de base de données

Partie A : Base de données

Le concepteur du site utilise une base de données simplifiée décrite par la représentation textuelle suivante :

Légende : Clé primaire | Clé étrangère 
Citoyens (cin, nom, prenom, email, telephone, dateNaissance, pin)
centres (id, nom, adresse, capaciteMax)
vaccins (id, nom, fabricant, stock)
reservations (cin #, iDcentre #, iDvaccin#, dateReservation, statut)

La description des différents champs est présentée dans le tableau suivant :

Champ Type (Taille) Description
cin CHAR(8) Numéro d'identité du citoyen
nom VARCHAR(50) Nom du citoyen
prenom VARCHAR(50) Prénom du citoyen
email VARCHAR(100) Adresse e-mail du citoyen
telephone VARCHAR(20) Numéro de téléphone du citoyen
dateNaissance DATE Date de naissance du citoyen
pin CHAR(6) Code PIN à 6 chiffres pour la connexion
id INT auto-incrémenté Identifiant unique du centre
nom VARCHAR(100) Nom du centre de vaccination
adresse VARCHAR(255) Adresse complète du centre
capaciteMax INT Nombre maximal de vaccinations par jour
id INT auto-incrémenté Identifiant unique du vaccin
nom VARCHAR(50) Nom du vaccin
fabricant VARCHAR(50) Laboratoire fabricant
stock INT Nombre de doses disponibles
dateReservation DATE Date prévue pour la vaccination
statut VARCHAR(20) État de la réservation ('confirmée', 'annulée', 'en attente')
Valeur par défaut : "en attente"

Tâches à réaliser :

  2. Importer dans la base crée prcédement le fichier "bd.sql"
  3. Ajouter la clé primaire ainsi que les trois références manquantes dans la table réservation.
  4. Exporter la base de données sous le même nom que celui mentionné dans la question n°1.

Étape 3: Création de la page reservation.html

2) Réservation d'un vaccin :

a) Créer la page "reservation.html" qui permet à un citoyen la réservation d'un vaccin via le formulaire suivant :
Informations personnelles
Informations de réservation
Centres de vaccination disponibles :
valuer contenu
1 Centre de vaccination Sbiba
2 Centre de vaccination Kasserine
3 Centre de vaccination Sbeitla
Types de vaccins disponibles :
valuer contenu
1 Vaxigrip
2 Influvac
3 Fluarix

Caractéristiques du formulaire :

  • Les zones de saisies de date de naissance et date de vaccination sont de type date.
  • La zone de saisie de l'email est de type mail.
  • La zone Pin est en lecture seul. Le pin est un code de 6 chiffres au hasard, ce code est généré dès le chargement de la page via une fonction JavaScript nommée "generatePin()".
  • Le clic sur le bouton "Enregistrer" fait appel à :
    • une fonction JavaScript nommée "verif1",
    • un fichier intitulé "reservation.php".
b) Développer dans le fichier "mesControles.js" la fonction JavaScript generatePin() permettant de générer et charger un code aléatoire de 6 chiffres dans la zone dédiée.
c) Développer, dans le fichier "mesControles.js", la fonction "Verif1" afin de s'assurer de la validité des champs du formulaire tout en respectant les contrôles suivants :
Champ Contrôle
CIN Chaîne de 8 chiffres
Nom Chaîne non vide de 30 caractères maximum commençant par une lettre.
Prénom Chaîne non vide de 30 caractères maximum commençant par une lettre.
Téléphone Chaîne de 8 chiffres dont le premier est supérieur strictement à 1.
Date de naissance Le choix d'une date est obligatoire.
Centre de vaccination Le choix d'un centre est obligatoire.
Type de vaccin Le choix d'un type est obligatoire.
Date de vaccination Supérieur à la date de système.
d) Créer le fichier "reservation.php" permettant de :
  • Afficher le message "CIN déjà utilisé !" si le CIN existe déjà dans la base de données.
  • Ou bien afficher le message "Centre complet !" si le centre de vaccination est saturé à la date choisie.
  • Ou bien afficher le message "Dose de vaccin épuisée !" si le stock de vaccins est insuffisant.
  • Ou bien insérer les informations du citoyen et les détails de la réservation dans les tables appropriées, puis afficher le message "Vaccination réservée avec succès !".

Suivi de réservation

Sachant que le clic sur le bouton "Enregistrer" fait appel à :

b) Développer, dans le fichier "mesControles.js", la fonction "verif2" permettant de s'assurer de la validité des champs du formulaire tout en respectant les contrôles suivants :
Champ Contrôle

c) Créer le fichier "suivie.php" permettant de :
  • Afficher le message "CIN inexistant !" si le CIN saisi n'est pas présent dans la base de données.

  • Ou bien afficher la fiche de vaccination suivante :

    Votre vaccin le plus proche sera :

    • Centre : [nom de centre]
    • Vaccin : [nom de vaccin]
